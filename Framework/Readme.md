# Product Framework

> Framework para Discovery, Estratégia, UX, Gestão, Construção e Evolução de Produtos Digitais.

O **Product Framework** é uma metodologia prática criada para organizar todo o ciclo de vida de um produto digital.

Seu objetivo é transformar uma ideia em um produto validado através de um processo estruturado, desde a descoberta do problema até a evolução contínua da solução.

Este Framework pode ser utilizado para documentar SaaS, MVPs, aplicativos, plataformas, sistemas internos ou qualquer outro produto digital.

---

# Objetivos

- Estruturar o processo de criação de produtos digitais.
- Padronizar a documentação dos projetos.
- Organizar Product Discovery, UX e Gestão de Produto.
- Facilitar a comunicação entre Produto, UX e Desenvolvimento.
- Criar Product Cases reutilizáveis para portfólio.

---

# Metodologia

Todo produto deverá seguir obrigatoriamente as seguintes etapas.

```text
Problema
      │
      ▼
01 - Discovery
      │
      ▼
02 - Strategy
      │
      ▼
03 - UX
      │
      ▼
04 - Product
      │
      ▼
05 - Build
      │
      ▼
06 - Growth
```

---

# Estrutura do Framework

## 01 - Discovery

Descobrir e validar o problema antes de pensar na solução.

---

## 02 - Strategy

Definir visão, posicionamento, proposta de valor e estratégia do produto.

---

## 03 - UX

Projetar a melhor experiência possível para o usuário.

---

## 04 - Product

Estruturar backlog, features, User Stories, critérios de aceite e documentação funcional.

---

## 05 - Build

Construir o MVP utilizando a stack definida.

---

## 06 - Growth

Medir resultados, validar hipóteses e evoluir continuamente o produto.

---

# Princípios

- Problemas antes de funcionalidades.
- Usuários antes de tecnologia.
- Validar antes de construir.
- Construir apenas o que gera valor.
- Documentar todas as decisões.
- Evoluir continuamente.
